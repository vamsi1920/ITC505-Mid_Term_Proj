Choose Your Own Adventure

Project Addendum

Process Overview

1. Planning and Storyboarding: I started by planning the story, mapping out the stages, and identifying potential choices that would lead to at least eight different endings. I created a flowchart to visualize the paths and endings, helping ensure the story was interactive and had meaningful choices.

2. HTML and CSS Structure: In the HTML, I created sections for displaying the story text, choices, and images. CSS was used to style these elements for a visually engaging layout. I wanted to make the UI clean and simple, so the user could focus on the story without distractions.

3. JavaScript Development: In JavaScript, I structured the story as an object with properties for text, choices, consequence, image, and link for each stage. Each choice triggered an update in the story's progression, creating a dynamic experience. Functions were built to start, update, and end the story, with updatePage() refreshing the content each time a choice was made.

Creative Choices

1. Story Concept: I chose a story about a cow and a tiger because it allowed for a wide range of scenarios â€” from light-hearted to suspenseful. Each choice aimed to create suspense or add humor, creating an immersive and engaging experience.

2. Visual Design: I wanted each ending to be visually distinct, so I sourced or created unique images to match each ending. I also added links to provide context or additional information for specific images, enhancing the interactive feel.

3. Choices and Consequences: To make the story more engaging, I crafted choices with varied consequences, leading to both positive and negative outcomes. This structure encouraged users to try different paths to see all possible endings, adding to the replay value.

Challenges Faced and Solutions

1. Image Paths on GitHub Pages: I initially faced issues with images not displaying correctly on GitHub Pages. I discovered that file paths were case-sensitive, and some images were incorrectly named. After correcting the names and confirming paths matched, the images displayed as expected.

2. Setting Image Links Dynamically: Adding clickable links to images in JavaScript required wrapping the image in an <a> tag and dynamically setting both the src for the image and href for the link. This added an extra layer of interactivity and solved the challenge of presenting additional content with each image.
